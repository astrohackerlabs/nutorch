# Experiment 1: The Astrohacker footer

## Description

Punch-list fix 1: the site's copyright becomes Astrohacker (linked to
astrohacker.com), using the HOUSE footer pattern researched across three of
Ryan's sites — shannon, termsurf, and keypears all ship the same element:

```html
<a href="https://astrohacker.com" class="flex items-center gap-2 hover:text-accent">
  <picture>
    <source srcset="/images/astrohacker-6-dark-32.webp 1x, /images/astrohacker-6-dark-64.webp 2x"
            media="(prefers-color-scheme: dark)" />
    <img src="/images/astrohacker-6-light-32.webp"
         srcset="/images/astrohacker-6-light-32.webp 1x, /images/astrohacker-6-light-64.webp 2x"
         alt="Astrohacker logo" class="h-5 w-5" />
  </picture>
  An Astrohacker Project
</a>
<p>&copy; {new Date().getFullYear()} Astrohacker. MIT License.</p>
```

Same assets everywhere (`astrohacker-6-{light,dark}-{32,64}.webp`, WebP,
committed directly — not generated by any pipeline), same alt text, same
wording. nutorch adopts the family signature with two deliberate deviations,
below.

**Decisions, made here:**

1. **The assets are copied from termsurf** (`website/public/images/` → ours),
   all four files, byte-for-byte (they are the v6 set; shasums recorded in the
   Result). They join `public/images/` as committed files — like the donor
   sites, NOT wired into `process-images.ts` (they are external brand assets,
   not derived from nutorch sources).
2. **Deviation 1 — the mode switch matches OUR theme system.** The donor sites
   switch logo variants with a `prefers-color-scheme` media query inside
   `<picture>`. nutorch's mode is `:root[data-theme]` (system pref + a TOGGLE
   persisted to localStorage — issue 0012 exp 1): an OS-media switch would
   desync from the toggle (light OS + dark toggle = wrong logo). So the footer
   renders BOTH `<img>` variants and the existing toggle-CSS pattern shows
   exactly one — the same mechanism as the sun/moon icons in `global.css`
   (`:root .toggle-moon { display: block }` etc.), extended with
   `.astrohacker-light`/`.astrohacker-dark` classes. Each img keeps the 1x/2x
   srcset and the canonical alt text.
3. **Deviation 2 — the license word stays truthful.** The donor sites say "MIT
   License."; nutorch is Apache-2.0 TODAY. The footer prints
   `© {year} Astrohacker · Apache-2.0` until punch-list fix 3 (the license
   change, its own experiment with an outward-facing tail) flips the word — this
   experiment does not get ahead of the license that is actually in the repo.
4. **Layout**: the footer keeps its two-row identity — row 1 as today (nutorch
   mark + name on the left, `github.com/nutorch/nutorch` on the right) minus the
   old copyright text; row 2 is the house Astrohacker block, centered: the
   linked logo + "An Astrohacker Project", then the copyright line. Final look
   judged by screenshot in both modes.
5. **Ryan X. Charles leaves the footer entirely** (the fix as stated: "copyright
   Astrohacker, not RXC").

## Changes

1. **`website/public/images/astrohacker-6-{light,dark}-{32,64}.webp`** (NEW, 4
   files): copied from termsurf, byte-identical.
2. **`website/src/components/Footer.astro`**: the new two-row footer.
3. **`website/src/styles/global.css`**: the
   `.astrohacker-light`/`.astrohacker-dark` toggle-display rules beside the
   existing sun/moon pattern.
4. **Nothing else** — no Rust, no docs content, no `v1/`.

1. **Build gate**: `bun run build` exits 0; the four webp assets land in
   `dist/images/`; `check:links` green (astrohacker.com joins the external-links
   list, never fetched).
2. **Asset fidelity**: shasums of the four copied files match termsurf's
   originals exactly.
3. **The toggle-correct switch, asserted**: built HTML contains BOTH logo
   variants with the toggle classes; the CSS rules exist; and a screenshot pair
   with `data-theme` pinned (the exp-1 harness) shows the LIGHT logo in light
   mode and the DARK logo in dark mode — on a page where the OS preference is
   irrelevant.
4. **Wording**: "An Astrohacker Project" links to https://astrohacker.com; the
   copyright line reads `© 2026 Astrohacker · Apache-2.0`; "Ryan X. Charles"
   appears nowhere in the built site (grep over `dist/`).
5. **Both modes, by eye**: landing-page footer screenshots, light and dark —
   house signature recognizable, on-brand, legible.
6. **Hygiene**: dprint clean on touched files; existing gates (`check:content`,
   `check:ops-ref`) untouched and green; `v1/` and the Rust tree untouched.

**Pass** = all six. **Fail** = the logo follows the OS instead of the toggle,
any RXC remnant in `dist/`, or asset drift from the donor set.
